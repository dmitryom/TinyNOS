    TinyNOS — Технический паспорт продукта¶

    Сетевая операционная система для AI-фабрики и кампусной сети¶

    Linux-native сетевая ОС для leaf-spine фабрик ИИ и кампусных сетей. Работает на стандартных whitebox-коммутаторах, без vendor lock-in.

    Релиз 1.0.0 · Лицензия Apache-2.0 · Схема конфигурации v34 · Исходный код: github.com/dmitryom/MultiCLI3

    Обзор¶

    TinyNOS — единый программный сетевой стек для двух классов сетей: leaf-spine фабрики ИИ (GPU-серверы и хранилище для обучения и инференса) и сети доступа / агрегации кампуса. Один образ, один CLI, одна модель автоматизации.

    Управление — через индустриальный CLI и программный интерфейс (YANG / OpenConfig / gNMI). Конфигурация транзакционная: изменение применяется целиком или откатывается. По объёму функций сопоставима с Cisco NX-OS, Arista EOS, NVIDIA Cumulus Linux, IP Infusion OcNOS и SONiC, оставаясь полностью открытой под Apache-2.0.

    Статусы зрелости в этом документе: GA — общедоступно и валидировано на программной / lab-фабрике · Preview — реализовано, аппаратная валидация в работе · Planned — в дорожной карте.

    Ключевые особенности¶

    • Одна NOS, две сети — один образ, CLI и автоматизация и для AI-фабрики, и для кампуса; один набор навыков и регламент эксплуатации.
    • Без vendor lock-in — открытое оборудование, открытый control plane, открытая лицензия. Смена вендора коммутатора без смены платформы ПО.
    • Безопасно by design — каждое изменение проходит транзакционный конвейер с автоматическим rollback; «частично применённой фабрики» не существует.
    • Стандарты в первую очередь — BGP, EVPN-VXLAN, OSPF, IS-IS, SRv6, VRRP, семейство spanning-tree и десятки RFC/IEEE для мультивендорного взаимодействия.
    • Создана для эксплуатации — метрики по каждому компоненту, структурированное логирование, append-only audit trail и RBAC штатно.
    • Честность по offload — функции с аппаратным ASIC-offload помечены реальной зрелостью; никаких выдуманных показателей масштаба.

    Программные функции¶

    Системные функции и управление конфигурацией¶

    Функция Описание Статус
    Транзакционная конфигурация Каждый commit валидируется, превьюится, применяется и самопроверяется как единое целое (6-стадийный конвейер) GA
    Автоматический rollback Любой сбой откатывает изменение начисто, без частичного состояния GA
    Commit-confirmed Изменения с дедлайном автоматически откатываются без подтверждения — безопасные удалённые правки GA
    Candidate / running / startup Раздельные хранилища конфигурации, validate, compare, diff, dry-run commit GA
    Checkpoints и revisions Именованные снапшоты и полная история изменений с атрибуцией оператора GA
    Сессии конфигурации Per-operator candidate с блокировкой, diff и отменой GA
    Модель данных YANG / OpenConfig Vendor-нейтральная, model-driven конфигурация (схема v34) GA

    L2 — Коммутация¶

    Функция Описание Статус
    VLAN 802.1Q Trunk / access / native VLAN, VLAN ID 1–4094 GA
    Q-in-Q (802.1ad) Provider bridging для сегментации оператора GA
    Spanning Tree (STP / RSTP / MSTP) Топологии без петель, multi-region MSTP, edge-порты, защита BPDU GA
    Агрегация каналов (LACP, 802.3ad) Бандлы static / LACP, несколько хеш-политик балансировки GA
    MLAG Двойно-активный top-of-rack с anycast VTEP и синхронизацией FDB GA
    LLDP / LLDP-MED Обнаружение соседей, voice-VLAN и network-policy конечным устройствам GA
    Обучение MAC и static FDB Динамическое обучение, статические записи моста и VXLAN GA
    Storm control Ограничение broadcast / multicast / unknown-unicast по kbps на порт GA
    Jumbo-фреймы MTU 64–9216 байт на интерфейсах, мостах и SVI GA

    Функция Описание Статус
    Статическая маршрутизация IPv4 / IPv6, UCMP-веса 1–255, resilient next-hop groups GA
    BGP-4 Multiprotocol, 4-байтный ASN, route-reflection, конфедерации, ADD-PATH, communities GA
    OSPFv2 Стандартная link-state маршрутизация IPv4 GA
    OSPFv3 Маршрутизация IPv6 Preview
    IS-IS Включая IPv6 multi-topology и TI-LFA fast-reroute GA
    RIPv2 Legacy distance-vector для brownfield-кампуса GA
    BFD Single- и multi-hop, субсекундное обнаружение отказов GA
    ECMP / UCMP Равно- и неравностоимостные пути, resilient hashing (buckets 32–4096) GA
    Multi-tenant VRF Жёсткая изоляция тенантов с контролируемым route leaking GA
    PBR Policy-based routing по правилам GA
    MPLS / SR-MPLS Сегментная маршрутизация по MPLS GA
    SRv6 (uSID, TI-LFA) IPv6-нативный segment routing с traffic engineering и fast-reroute GA

    Функция Описание Статус
    PIM Sparse-Mode Эффективная доставка «один-ко-многим» GA
    IGMP / MLD snooping Querier и контроль multicast-членства на мосту GA
    MSDP Anycast-RP source discovery между доменами GA

    Функция Описание Статус

    ACL и безопасность¶

    Функция Описание Статус
    ACL (IPv4 / IPv6) Stateless-фильтрация по L3/L4, временные окна, QoS-ремаркинг (движок nftables) GA
    NAT (source-NAT) Трансляция адресов источника GA
    Control-plane policing (CoPP) Защита CPU коммутатора с курируемым профилем по умолчанию GA
    First-hop security DHCP snooping, Dynamic ARP Inspection, IP Source Guard GA
    RBAC Авторизация команд по ролям на каждой операции GA
    TACACS+ / RADIUS Централизованная аутентификация, авторизация и accounting GA
    Защита секретов Учётные данные скрыты из вывода и хранятся защищёнными «в покое» (one-way hash для логинов) GA
    Audit trail Защищённый от подмены append-only журнал изменений и решений о доступе GA
    MACsec (802.1AE) Шифрование на канальном уровне для доверенных линков Planned

    Масштаб и характеристики системы¶

    Указаны реальные, заданные кодом пределы. Параметры, зависящие от ASIC коммутатора (размер MAC-таблицы, FIB/RIB-ёмкость, число одновременных VXLAN-туннелей), определяются платформой и публикуются по мере whitebox bring-up — выдуманных аппаратных цифр в этом документе нет.

    Параметр Значение
    VLAN ID 1–4094
    MTU кадра 64–9216 байт
    BGP ASN 4-байтный (1–4 294 967 295)
    DSCP remarking 0–63
    802.1p приоритет 0–7
    Административная дистанция (static) 1–255
    ECMP resilient buckets 32–4096 (степень двойки)
    VXLAN VNI 1–16 777 215 (24-бит, по стандарту)
    Версия схемы конфигурации v34
    Поверхность CLI 850+ команд set / show
    Готовых lab-сценариев 66 (EVPN, MLAG, SRv6, ECMP, lossless QoS и др.)
    MAC / FIB / RIB / ARP-ND ёмкость Зависит от платформы — публикуется при bring-up оборудования

    Соответствие стандартам¶

    Маршрутизация: BGP-4 (RFC 4271) с multiprotocol, route-reflection, конфедерациями, 4-байтным ASN, graceful restart, ADD-PATH и standard/extended/large communities · OSPFv2 · OSPFv3 (Preview) · IS-IS, вкл. IPv6 multi-topology и TI-LFA · RIPv2 · BFD single- и multi-hop.

    Фабрика и оверлей: EVPN (RFC 7432) · EVPN-VXLAN (RFC 8365) · EVPN IRB (RFC 9135) · EVPN IP-Prefix (RFC 9136) · VXLAN (RFC 7348).

    Segment routing: SR-архитектура (RFC 8402) · SRv6 SRH (RFC 8754) · SRv6 network programming (RFC 8986) · BGP overlay для SRv6 (RFC 9252).

    Multicast: PIM Sparse-Mode (RFC 7761) · MSDP (RFC 3618).

    Коммутация: 802.1Q VLAN · 802.1D/w/s spanning tree · 802.1AX link aggregation · 802.1AB LLDP · IGMP/MLD snooping.

    QoS и lossless: 802.1Qbb PFC (Preview) · 802.1Qaz ETS/DCBX (Preview) · ECN (RFC 3168).

    Высокая доступность: VRRP v2/v3 (RFC 5798).

    Безопасность: MACsec 802.1AE (Planned) · SNMP v3 framework (Preview).

    Управление: YANG 1.0/1.1 (RFC 6020 / 7950) · модели OpenConfig · gNMI (gRPC + HTTP/JSON, вкл. Subscribe) · NETCONF base (Preview) · NTPv4 · PTP v2 (IEEE 1588) (Preview).

    Поддерживаемые платформы¶

    Сегодня (v1.0.0): программная и виртуальная dataplane production-класса — bare-metal x86, KVM и lab-фабрики containerlab / OrbStack. Для валидации дизайна, разработки автоматизации, симуляции сети и обучения.

    В bring-up (v1.1): открытые whitebox-коммутаторы на индустриальных ASIC —

    Платформа ASIC Плотность портов
    Edgecore AS9516-32D Broadcom Tomahawk 3 32 × 100GbE
    Edgecore AS7726-32X Broadcom Trident 4 32 × 400GbE
    Celestica DS3000 Broadcom Trident 3 32 × 100GbE
    UfiSpace S9600-72XC Broadcom Tomahawk 2 72 × 100GbE
    NVIDIA Mellanox SN3700 NVIDIA Spectrum-3 32 × 200GbE
    NVIDIA Mellanox SN4600C NVIDIA Spectrum-4 64 × 200GbE

    Примечание о зрелости (прочитайте это): TinyNOS сегодня общедоступен как программная и валидированная в lab NOS. Bring-up на whitebox-оборудовании — мониторинг здоровья платформы, подписанный образ установщика и аппаратный ASIC-offload — в активной разработке для v1.1. Аппаратный offload дейтаплейна требует поставляемого вендором драйвера коммутатора и пока не является общедоступным. Каждая возможность выше помечена своей реальной зрелостью; аппаратные показатели масштаба по платформам будут опубликованы по мере завершения bring-up.
