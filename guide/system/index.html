    Настройка системы¶

    Режимы CLI и навигация¶

    TinyNOS разделяет два режима. В операционном режиме доступны команды просмотра и диагностики, приглашение оканчивается на >. В конфигурационном режиме вносятся правки, приглашение оканчивается на #.

    Чтобы войти в конфигурационный режим, выполните configure.

    Действие Команда
    Контекстная справка по доступным командам ?
    Автодополнение текущего слова Tab
    Выход из текущего режима на уровень выше exit или quit
    Возврат в операционный режим end 
    leaf1> configure
leaf1#

    Модель конфигурации¶

    Конфигурация хранится в трёх уровнях:

    Уровень Содержимое
    Candidate Рабочая копия. В неё вносятся правки командами set и delete
    Running Конфигурация, применённая сейчас
    Startup Загрузочная конфигурация, восстанавливается после перезагрузки

    Применение транзакционное. При commit candidate проходит проверку, рендеринг, предварительный контроль (preflight), применение и постпроверку. Если любая из стадий завершается ошибкой, изменение откатывается целиком, и узел остаётся на прежней конфигурации.

    Сессии и блокировки¶

    Чтобы работать в именованной сессии конфигурации, выполните:

    leaf1# configure session maintenance-2026

    Чтобы взять исключительную блокировку на правку конфигурации, выполните:

    leaf1# lock config

    Применение и откат¶

    Чтобы проверить candidate перед применением, выполните validate. Чтобы увидеть отличия candidate от running, выполните compare candidate running.

    Команда Назначение
    validate Проверить candidate на ошибки
    compare candidate running Показать отличия рабочей копии от применённой конфигурации
    commit Применить candidate
    commit confirmed <min> Применить с таймером автоотката на <min> минут
    confirm Подтвердить применение, начатое через commit confirmed
    rollback Откатить последний commit к предыдущему состоянию running
    rollback to <checkpoint> Вернуться к именованной контрольной точке
    save Записать текущую конфигурацию как загрузочную
    show blockers Показать причины, по которым commit будет отклонён
    show checkpoints Список контрольных точек
    show revisions История применённых конфигураций

    Система¶

    Чтобы задать имя узла, выполните:

    leaf1# set system hostname leaf1

    Чтобы задать доменное имя, выполните:

    leaf1# set system domain-name dc1.example.net

    AAA и RBAC¶

    Роли определяют разрешённые команды и параметры shell. Пользователи привязываются к ролям.

    Чтобы создать роль и разрешить ей шаблон команд, выполните:

    leaf1# set aaa role netops permit "show *"
leaf1# set aaa role netops shell /bin/bash privilege 15

    Чтобы создать локального пользователя, выполните:

    leaf1# set aaa user operator password S3cretPass role netops
leaf1# set aaa user operator ssh-key "ssh-ed25519 AAAA... operator@host"

    Чтобы задать порядок методов аутентификации, выполните:

    leaf1# set aaa authentication order local,tacacs,radius

    Чтобы добавить серверы TACACS+ и RADIUS, выполните:

    leaf1# set aaa tacacs server tac1 host 10.1.0.10 key SharedKey timeout 5
leaf1# set aaa radius server rad1 host 10.1.0.20 secret SharedSecret timeout 5

    Системный backend задаёт порядок источников NSS и сопоставление авторизации:

    leaf1# set aaa system-backend nss passwd-order files,sss,ldap
leaf1# set aaa system-backend nss group-order files,sss,ldap
leaf1# set aaa system-backend nss shadow-order files,sss,ldap
leaf1# set aaa system-backend authorization map user alice role netops
leaf1# set aaa system-backend authorization map group netadmins role netops
leaf1# set aaa system-backend accounting local enable true
leaf1# set aaa system-backend accounting remote mode tacacs

    Защита от блокировки администратора

    Оставляйте local в authentication order и хотя бы одного локального пользователя с паролем или SSH-ключом. Иначе при недоступности TACACS+ или RADIUS администратор будет заблокирован, и вход на узел станет невозможен.

    Control-plane policing (CoPP)¶

    CoPP ограничивает трафик, направляемый к процессору узла, по классам. Скорость задаётся в пакетах в секунду (pps).

    Чтобы развернуть предопределённый набор классов по умолчанию (маршрутизация, OAM, управление, L2, критичный трафик) не затрагивая уже настроенные классы, выполните:

    leaf1# set copp profile default

    Чтобы задать собственный класс и привязать к нему протокол, выполните:

    leaf1# set copp class oam rate 1000 burst 2000
leaf1# set copp class oam protocol icmp

    Чтобы посмотреть текущие классы CoPP, выполните show copp.

    Списки доступа (ACL)¶

    ACL состоит из правил, нумерованных параметром <seq>; правила проверяются по возрастанию <seq>, применяется первое совпавшее. Трафик, не совпавший ни с одним правилом, пропускается — добавьте завершающее правило deny, если нужен явный запрет. Имена ACL регистронезависимы. Привязка ACL к интерфейсу задаёт направление обработки in или out.

    Чтобы создать правила фильтрации, выполните:

    leaf1# set acl name web 10 permit
leaf1# set acl name web 20 permit proto tcp
leaf1# set acl name web 30 permit src 10.0.0.0/24
leaf1# set acl name web 40 deny dst 192.168.0.0/16
leaf1# set acl name web 50 permit proto tcp src-port 1024
leaf1# set acl name web 60 permit proto tcp dst-port 443

    src-port и dst-port учитываются только вместе с proto tcp или proto udp.

    Правило может переписывать поля проходящих кадров и пакетов:

    leaf1# set acl name web 70 permit set dscp 46
leaf1# set acl name web 80 permit set priority 5
leaf1# set acl name web 90 permit set vlan 100

    Правило может ограничиваться временным окном, журналироваться и считаться:

    leaf1# set acl name web 100 permit time 08:00-20:00
leaf1# set acl name web 110 deny log
leaf1# set acl name web 120 permit count

    Чтобы привязать ACL к интерфейсу с направлением, выполните:

    leaf1# set interfaces eth1 acl in web
leaf1# set interfaces eth1 acl out web

    Чтобы посмотреть ACL, выполните show acl или show acl name web.
