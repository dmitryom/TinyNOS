    Сетевая виртуализация¶

    Оверлей (наложенная сеть) строится поверх настроенного L3-андерлея, см. Layer 3.

    EVPN и VTEP¶

    Источник VTEP — loopback, анонсированный в underlay. Чтобы задать источник и включить EVPN-сигнализацию, выполните:

    leaf1# set evpn vtep source-ip 10.0.0.1
leaf1# set evpn vtep source-interface lo
leaf1# set routing bgp evpn neighbor 10.0.0.254 activate
leaf1# set evpn advertise-all-vni enable

    Для IPv6-underlay задайте IPv6-источник VTEP:

    leaf1# set evpn vtep source-ipv6 2001:db8::1

    Чтобы отключить анонс всех VNI, используйте disable:

    leaf1# set evpn advertise-all-vni disable

    Состояние EVPN и VTEP:

    leaf1> show evpn

    VXLAN-устройства¶

    UDP-порт VXLAN по умолчанию 4789. Чтобы изменить порт, выполните:

    leaf1# set evpn vxlan udp-port 4789

    Режим single-vxlan-device (SVD) использует одно VXLAN-устройство для всех VNI. Чтобы включить SVD и задать его параметры, выполните:

    leaf1# set evpn single-vxlan-device enable true
leaf1# set evpn single-vxlan-device name vxlan0
leaf1# set evpn single-vxlan-device vni 4096
leaf1# set evpn single-vxlan-device dataplane-policy auto

    Параметр vni задаёт базовый VNI самого устройства SVD — его не следует путать с L2-VNI отдельных VLAN (см. ниже). Политика dataplane-policy принимает значения auto, native, compat-emulated, external. По умолчанию используйте auto — оно выбирает реализацию по возможностям ядра. Остальные значения — явные переопределения: native — нативный vnifilter (ядро ≥ 5.18), compat-emulated — отдельное VXLAN-устройство на каждый VNI для старых ядер, external — устройство создаётся вне TinyNOS.

    Чтобы отключить SVD, выполните:

    leaf1# set evpn single-vxlan-device enable false

    Атрибуты инкапсуляции VXLAN задаются отдельно. Чтобы задать MTU, режим MTU, TTL, ToS и L2-обучение, выполните:

    leaf1# set evpn vxlan mtu 9000
leaf1# set evpn vxlan mtu-mode explicit
leaf1# set evpn vxlan ttl 64
leaf1# set evpn vxlan ttl inherit
leaf1# set evpn vxlan tos inherit
leaf1# set evpn vxlan learning off

    Режим mtu-mode auto выводит MTU из source-устройства. ttl inherit наследует TTL внутреннего пакета. При управлении через EVPN обучение MAC в плоскости данных должно быть выключено (learning off) — за изучение адресов отвечает control plane; в нативном и legacy-режимах допускается learning on.

    Ядро для SVD

    Режим single-vxlan-device (SVD) требует ядра Linux ≥ 5.18.

    VLAN → VNI (L2-VNI)¶

    L2-VNI связывает VLAN с VNI и переносит широковещательный домен через фабрику. Привязать VLAN к VNI можно любой из двух эквивалентных команд:

    leaf1# set evpn vlan 10 vni 10010
leaf1# set vlan-map vlan 10 vni 10010

    Чтобы включить подавление ARP и задать gateway-MAC для L2-VNI, выполните:

    leaf1# set evpn l2-vni 10 arp-suppression enable
leaf1# set evpn l2-vni 10 gateway-mac 00:11:22:33:44:55

    Чтобы отключить подавление ARP, используйте disable:

    leaf1# set evpn l2-vni 10 arp-suppression disable

    Хост-порт подключается к мосту в режиме access на нужный VLAN:

    leaf1# set interfaces eth3 bridge br-default
leaf1# set interfaces eth3 l2 mode access vlan 10

    Детали по VNI:

    leaf1> show evpn vni detail

    Маршрутизация VXLAN и symmetric IRB¶

    Маршрутизация между VNI выполняется через L3-VNI и VRF. Чтобы создать L3VRF и привязать L3-VNI, выполните:

    leaf1# set network-instances tenant-red type l3vrf
leaf1# set evpn l3-vni 5000 vrf tenant-red
leaf1# set evpn irb-mode symmetric

    Режим irb-mode принимает symmetric или asymmetric. В symmetric-IRB маршрутизация на ingress- и egress-leaf использует общий транзитный L3-VNI (route-and-bridge на обоих узлах). В asymmetric-IRB ingress-leaf маршрутизирует сразу в L2-VNI назначения, без транзитного L3-VNI, поэтому каждый leaf должен знать все целевые подсети.

    Anycast-шлюз использует общий MAC на всех leaf. Чтобы задать anycast-MAC, выполните:

    leaf1# set evpn anycast-gateway mac 00:00:5e:00:01:01

    SVI несёт IP-адрес шлюза и виртуальный MAC:

    leaf1# set interfaces vlan 10 ipv4 address 10.10.10.1/24
leaf1# set interfaces vlan 10 mac virtual 00:00:5e:00:01:01

    Чтобы анонсировать default-gateway и SVI-IP через EVPN, выполните:

    leaf1# set evpn route-advertise default-gateway on
leaf1# set evpn route-advertise svi-ip on

    Для отключения анонса используйте off:

    leaf1# set evpn route-advertise svi-ip off

    BUM-репликация¶

    BUM-трафик (broadcast, unknown-unicast, multicast) реплицируется одним из двух методов. Чтобы выбрать head-end replication или multicast-group, выполните:

    leaf1# set evpn bum-replication head-end-replication
leaf1# set evpn bum-replication multicast-group

    Для multicast-group задайте группу и uplink-интерфейс:

    leaf1# set evpn vxlan multicast-group 239.1.1.1
leaf1# set evpn vxlan multicast-uplink-interface eth1

    Multicast-group требует группы

    Режим multicast-group требует заданной multicast-group, иначе BUM-трафик (ARP/ND) теряется по фабрике.

    EVPN multihoming (all-active)¶

    EVPN multihoming подключает хост к двум leaf в режиме all-active через Ethernet Segment. Чтобы настроить сегмент на интерфейсе, выполните:

    leaf1# set interfaces eth4 evpn mh es-id 1
leaf1# set interfaces eth4 evpn mh es-sys-mac 00:11:22:00:00:01
leaf1# set interfaces eth4 evpn mh es-df-pref 100
leaf1# set interfaces eth4 evpn mh uplink

    Значение es-id одинаковое на обоих коммутаторах сегмента. При выборе designated-forwarder (DF) для Ethernet Segment предпочтение отдаётся узлу с бо́льшим значением es-df-pref. Не назначайте ESI на бонд peer-link MLAG.

    Статические VXLAN-туннели¶

    Без EVPN-сигнализации peer-ы задаются вручную через статическую FDB для head-end replication. Чтобы добавить запись, выполните:

    leaf1# set bridge br-default static-fdb mac 00:11:22:33:44:66 port vxlan0 dst 10.0.0.2 vlan 10 src_vni 10010

    Параметр src_vni необязателен и указывает VNI исходного туннеля.

    Проверка¶

    Сводка EVPN-сессий и анонсированных маршрутов в BGP:

    leaf1> show bgp l2vpn evpn summary
leaf1> show bgp l2vpn evpn route
