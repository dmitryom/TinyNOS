    Layer 3¶

    Маршрутизация TinyNOS включает статические маршруты, ECMP, BGP, OSPF, IS-IS, BFD, VRF, PIM, NAT и статические соседей. Команды задают намерение в кандидат-конфигурации; примените их через commit.

    Статическая маршрутизация¶

    Добавьте статический маршрут или blackhole:

    leaf1# set routing static route 10.20.0.0/16 next-hop 10.0.0.2
leaf1# set routing static route 192.0.2.0/24 blackhole

    Проверьте таблицу маршрутизации и соседей:

    leaf1> show ip route
leaf1> show ip route 10.20.0.0/16
leaf1> show ip neighbors

    ECMP¶

    Равностоимостные пути балансируются автоматически. Для приёма нескольких путей через BGP включите multipath и ослабьте сравнение AS-path:

    leaf1# set routing bgp maximum-paths 8
leaf1# set routing bgp bestpath as-path multipath-relax

    BGP¶

    Задайте локальную AS, router-id и соседа с удалённой AS:

    leaf1# set routing bgp local-as 65001
leaf1# set routing bgp router-id 10.0.0.1
leaf1# set routing bgp neighbor 10.0.0.2 remote-as 65002

    Сгруппируйте соседей в peer-group с общими параметрами:

    leaf1# set routing bgp peer-group FABRIC remote-as 65002
leaf1# set routing bgp neighbor 10.0.0.2 peer-group FABRIC

    Настройте источник обновлений, next-hop-self и BFD для сессии:

    leaf1# set routing bgp neighbor 10.0.0.2 update-source lo
leaf1# set routing bgp neighbor 10.0.0.2 next-hop-self
leaf1# set routing bgp neighbor 10.0.0.2 bfd

    Задайте таймеры, аутентификацию через ссылку на секрет и политику route-map:

    leaf1# set routing bgp neighbor 10.0.0.2 timers keepalive 3 hold-time 9
leaf1# set routing bgp neighbor 10.0.0.2 password-ref BGP_FABRIC
leaf1# set routing bgp neighbor 10.0.0.2 route-map-in IMPORT
leaf1# set routing bgp neighbor 10.0.0.2 route-map-out EXPORT

    Назначьте соседа клиентом route-reflector и анонсируйте префикс:

    leaf1# set routing bgp neighbor 10.0.0.2 route-reflector-client
leaf1# set routing bgp network 10.10.0.0/24

    Включите graceful-restart с сохранением FIB на время рестарта control-plane:

    leaf1# set routing bgp graceful-restart
leaf1# set routing bgp graceful-restart preserve-fw-state

    Для BGP unnumbered соседа задают по имени интерфейса, а у peer-group указывают remote-as external:

    leaf1# set routing bgp peer-group FABRIC remote-as external
leaf1# set routing bgp neighbor eth1 peer-group FABRIC

    Проверьте состояние сессий:

    leaf1> show bgp summary
leaf1> show bgp neighbors

    OSPF¶

    Задайте router-id, привяжите сеть к area и определите тип area:

    leaf1# set routing ospf router-id 10.0.0.1
leaf1# set routing ospf network 10.10.0.0/24 area 0
leaf1# set routing ospf area 0.0.0.1 type stub
leaf1# set routing ospf area 0.0.0.2 type nssa

    Перераспределите маршруты, исключите интерфейс из расчёта и включите MD5-аутентификацию:

    leaf1# set routing ospf redistribute connected
leaf1# set routing ospf redistribute static
leaf1# set routing ospf redistribute bgp
leaf1# set routing ospf passive-interface eth2
leaf1# set routing ospf interface eth1 auth md5 key-id 1 key S3cret

    Включите BFD, graceful-restart и ширину ECMP:

    leaf1# set routing ospf bfd
leaf1# set routing ospf graceful-restart
leaf1# set routing ospf maximum-paths 8

    IS-IS¶

    Включите TI-LFA fast-reroute и задайте режим защиты и уровень:

    leaf1# set routing isis fast-reroute ti-lfa enable true
leaf1# set routing isis fast-reroute ti-lfa mode node-protection
leaf1# set routing isis fast-reroute ti-lfa level all

    Задайте ширину ECMP и проверьте смежности:

    leaf1# set routing isis maximum-paths 8
leaf1> show isis adjacency

    Чтобы отключить TI-LFA или ограничить его одним уровнем:

    leaf1# set routing isis fast-reroute ti-lfa enable false
leaf1# set routing isis fast-reroute ti-lfa mode link-protection
leaf1# set routing isis fast-reroute ti-lfa level level-1

    BFD¶

    Привяжите BFD-сессию к протоколу и задайте интервалы. Параметры min-tx и min-rx указываются в миллисекундах; multiplier — число пропущенных пакетов до объявления отказа.

    leaf1# set bfd peer 10.0.0.2 protocol bgp min-tx 300 min-rx 300 multiplier 3
leaf1# set bfd peer 10.0.0.3 protocol ospf min-tx 300 min-rx 300 multiplier 3
leaf1# set bfd peer 10.0.0.4 protocol static min-tx 300 min-rx 300 multiplier 3

    Проверьте состояние сессий:

    leaf1> show bfd

    VRF¶

    Создайте L3-VRF, привяжите интерфейс и задайте route-distinguisher:

    leaf1# set network-instances RED type l3vrf
leaf1# set vrf RED interface eth3
leaf1# set vrf RED rd 65001:100

    Настройте утечку маршрутов между VRF через route-target:

    leaf1# set vrf RED route-leak import 65001:200
leaf1# set vrf RED route-leak export 65001:100

    Запустите BGP в контексте VRF:

    leaf1# set routing bgp vrf RED local-as 65001
leaf1# set routing bgp vrf RED neighbor 10.30.0.2 remote-as 65010
leaf1# set routing bgp vrf RED network 10.30.0.0/24
leaf1# set routing bgp vrf RED redistribute connected
leaf1# set routing bgp vrf RED advertise ipv4 unicast

    PIM (multicast)¶

    Включите graceful-restart для PIM и настройте MSDP-пира с указанием локального источника:

    leaf1# set routing pim graceful-restart
leaf1# set routing pim msdp-peer 10.0.0.5 source 10.0.0.1

    NAT¶

    Настройте source-NAT через masquerade (адрес выходного интерфейса) или фиксированный адрес:

    leaf1# set nat source OUTBOUND out-interface eth0 masquerade
leaf1# set nat source OUTBOUND match 10.10.0.0/24
leaf1# set nat source FIXED out-interface eth0 address 203.0.113.10

    Настройте destination-NAT (проброс порта) с заменой адреса и при необходимости порта:

    leaf1# set nat destination WEB in-interface eth0 proto tcp dport 80 to 10.10.0.10
leaf1# set nat destination SSH in-interface eth0 proto tcp dport 2222 to 10.10.0.10 port 22

    Статические соседи (ARP / ND)¶

    Задайте статическую запись ARP или ND для адреса на конкретном интерфейсе:

    leaf1# set routing neighbor 10.10.0.50 mac 00:11:22:33:44:55 dev eth1
